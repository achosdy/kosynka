# Пасьянс «Косынка» — локальная PWA

Готовая статическая игра для GitHub Pages. Проект не требует сборщика, сервера, базы данных, CDN или сторонних сервисов: достаточно загрузить содержимое папки в репозиторий.

## Возможности

- классическая Косынка: семь столбцов, четыре основания, раздача по одной карте и две пересдачи;
- счётчик ходов, таймер и лучший результат в `localStorage`;
- новая игра, перезапуск исходной раздачи, отмена до 150 ходов и подсказка;
- управление кликом/касанием и перетаскиванием мышью;
- автоматические состояния **You won** и **Game over**;
- тринадцать тем: Default, Dark, Light, Winx, Pokémon, Minecraft, Halloween, Christmas, Formula 1, Hello Kitty, Pirates, Чернобыль и Mario;
- индивидуальные символы мастей, персонажи карт, фон, анимация и справка для каждой темы;
- адаптация под Windows, macOS и iPhone;
- PWA-манифест, Service Worker и офлайн-запуск после первого открытия;
- кнопка установки, отдельная инструкция для Safari на iPhone и запасная загрузка ZIP.

## Публикация на GitHub Pages

1. Создайте новый публичный или приватный репозиторий.
2. Загрузите **всё содержимое этой папки в корень репозитория**, сохранив структуру `assets/` и `downloads/`.
3. Откройте **Settings → Pages**.
4. В разделе **Build and deployment** выберите **Deploy from a branch**.
5. Выберите ветку `main`, папку `/ (root)` и нажмите **Save**.
6. Откройте выданный GitHub Pages URL. Service Worker и установка PWA работают через HTTPS.

Все пути относительные, поэтому проект корректно работает и в пользовательском репозитории вида `https://username.github.io/repository/`.

## Локальный запуск

Service Worker не запускается при прямом открытии `file://`. Для полноценной проверки используйте любой локальный HTTP-сервер:

```bash
python -m http.server 8000
```

Затем откройте `http://localhost:8000`.

## Иконки

- Windows: `assets/icons/kosynka.ico`, `assets/icons/windows-icon-256.png`;
- macOS: `assets/icons/kosynka.icns`, `assets/icons/macos-icon-512.png`;
- iOS: `assets/icons/apple-touch-icon.png`;
- PWA: PNG 192/256/512 и отдельная maskable-иконка.

## Главные файлы

- `index.html` — интерфейс и окна;
- `styles.css` — адаптивная вёрстка, темы и анимации;
- `app.js` — игровая логика без библиотек;
- `manifest.webmanifest` — установка приложения;
- `sw.js` — офлайн-кэш;
- `downloads/kosynka-pwa.zip` — архив, который скачивается по кнопке внутри игры.

## Замена изображений тем

Фоны находятся в `assets/backgrounds/`, миниатюры — в `assets/thumbs/`. Можно заменить файл, сохранив имя и формат WebP. Внешние ссылки не используются.

## Права на тематические материалы

В проект включены локальные изображения, предоставленные заказчиком. Перед публичной или коммерческой публикацией тем по известным франшизам владелец репозитория должен самостоятельно проверить права на изображения, названия и персонажей. Игровой код и базовая иконка не зависят от этих изображений.
